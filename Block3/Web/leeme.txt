El sitio web desarrollado simula la que sería la web oficial de un estudio de videojuegos 
creado por mí y unos amigos. 
Cabe destacar que la mayoría de enlaces relacionados con redes sociales del estudio o sus 
miembros no llevan a ninguna parte para no exponer sus datos.


En cuanto a HTML, se ha procurado usar sus etiquetas semánticas. Las páginas cuentan con un header que detalla su idioma, título y hojas de estilo asociadas.
El cuerpo ("body") de la página lo compone básicamente un "wrapper". Este wrapper usa CSS grid para comprender el contenido de la página:
	-> La primera fila del grid está dedicada al "nav".
	-> La última fila del grid está dedicada al "footer".
	-> Las filas intermedias se dedican al contenido de la página. Cada bloque de contenido con entidad se enmarca en una etiqueta "section" y, en el caso de la página
	de inicio, existe una etiqueta "aside" que enmarca dos (o más) noticias o novedades destacadas del estudio.

Siguiendo con el CSS, la web cuenta con dos tipos de páginas:
	-> Las que son como la home, con contenido principal + noticias a un lateral
	-> El resto, con contenido principal centrado y sin noticias al lateral 
	
Se han creado dos ficheros CSS (base.css y base-3-rows.css) que sirven de base para maquetar futuras páginas para cada uno de los dos tipos. 
Los estilos del nav, footer y particularidades de cada sub-página también se han apartado a CSS menores distinguibles por su nombre.
Se ha evitado en uso de ids y favorecido el de clases.

Las hojas de estilos se valen de media queries para hacer la web responsive según el dispositivo de visualización. Destacan los cambios en:
	-> Tamaños de fuente y márgenes de los textos.
	-> La ubicación del nav, que pasa de ser un menú en la parte superior a un menú desplegable en tablets y móviles.
	-> La página de inicio, que reubica la sección lateral de novedades.
	-> La página de "equipo", que cambia la forma de enseñar las tarjetas de los miembros del equipo.

Más detalles de la web son los siguientes:
	-> No se ha empleado niguna herramienta como Bootstrap o similares.
	-> Se usa CSS grid para maquetar el cuerpo de las páginas.
	-> Se usa en varios casos la utilidad "flexbox" para mostrar el contenido de forma conveniente (véase las novedades de la página principal, 
	las tarjetas de los miembros del estudio o el footer). Se suele cambiar la propiedad "flex-direction" de "row" a "column" cuando se sirve la web a tablets y móviles.
	-> Se hace uso de "iframe" para incrustar contenido externo a la web, como el trailer de un videojuego en la sección "objetivos" o un mapa personalizado en "contacto".
	-> Se ha hecho uso de la herramienta "Font Awesome" para incluir iconos en la página web.



Todos los documentos HTML y CSS han sido validados en los validadores del W3C sin advertencias. 
Así mismo, se han pasado los comparadores de accesibilidad, comprobando la gravedad de los potenciales problemas y corrigiendo 
los problemas encontrados (destaca que la web ha pasado la validación AA y AAA de achecker).
